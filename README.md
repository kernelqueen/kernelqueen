I make DSA and LeetCode walkthroughs on YouTube, primarily in **Java**. Most of what I publish here is the working code behind those videos solutions, patterns, and the curated problem lists I use when I prep.

## On YouTube

Over on [**@kernel-queen**](https://www.youtube.com/@kernel-queen), I walk through LeetCode and DSA problems in Java - focusing on the *why* behind each step, not just the final code. If you learn best by watching someone reason through a problem out loud, the channel is built for you.

## Philosophy

> I'm here for the slow, stubborn version of growth - one problem a day, one pattern at a time. The repo is the receipts; the channel is the working out.

I lean toward **patterns over memorization**, **explanations over shortcuts**, and **showing up daily** even when the progress is invisible.
